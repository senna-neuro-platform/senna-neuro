# ADR-0008: Template Usage Policy

Status: Accepted  
Date: 2026-03-07

## Context

Для C++-ядра проекта важно использовать zero-overhead подход: повторно использовать логику без лишнего runtime-polymorphism и лишних аллокаций, особенно на горячем пути симуляции.

## Decision

1. Шаблоны применяются там, где есть вариативность типов данных или стратегий, влияющая на производительность/расширяемость.
2. Для дефолтного сценария всегда задаются понятные alias/конкретные типы (например, `Coord3D`), чтобы не усложнять API.
3. Сложные метапрограммные конструкции без доказанной пользы не вводятся.
4. Runtime-полиморфизм оставляется для границ интеграции, а не для внутреннего hot path, где это можно выразить templates.

## Consequences

- Ядро получает расширяемость без накладных расходов виртуальных вызовов.
- Код остается читаемым за счет явных alias и ограниченного объема шаблонной логики.
- Новые шаблонные элементы добавляются только при наличии явного инженерного обоснования.
